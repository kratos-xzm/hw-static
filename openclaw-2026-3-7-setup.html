踩坑实录：升级过程中的 3 个大坑

在正式说配置之前，先说说我都踩了哪些坑。这些坑你要是提前知道，能省至少 1 个小时。

坑 1：模型配置格式变了

旧版本的模型配置是这样的：

{
  "model": "gpt-4",
  "api_key": "sk-xxx"
}

2026.3.7 版本改成了嵌套结构：

{
  "models": {
    "default": {
      "provider": "openai",
      "model": "gpt-5.4",
      "api_key": "sk-xxx",
      "base_url": "https://api.openai.com/v1"
    }
  }
}

如果你不改格式，启动直接报错。报错信息还特别迷惑，说什么"provider not found"，我一度以为是网络问题。

坑 2：Docker 构建失败了 5 次

新版精简了 Docker 多阶段构建，听起来是好事，但实际上...

如果你之前用过旧版 Docker 镜像，直接 docker pull 更新可能会遇到层缓存问题。我的解决方案是：

# 彻底清理旧镜像
docker rmi $(docker images -q "*openclaw*") -f
docker builder prune -f

# 重新构建
docker build --no-cache -t openclaw:2026.3.7 .

坑 3：GPT-5.4 国内无法直接访问

这是最大的问题。

GPT-5.4 目前只通过 OpenAI 官方 API 提供，国内直接访问会被...你懂的。即使你有 API Key，请求也会超时或直接被拒绝。

我试了几种方案：

  • 自己搭代理？太麻烦，还要维护
  • 用第三方转发服务？怕不稳定，也担心安全问题
  • 换 Claude？但是 GPT-5.4 的电脑操作能力真的香啊

最后我用了一个叫 KL-API 的中转方案，算是目前最省心的选择了。

实战配置：接入 GPT-5.4 的完整步骤

好了，下面是实操环节。

我假设你已经安装好了 OpenClaw 2026.3.7，如果还没安装，可以参考官方文档或者阿里云的部署教程（链接我放文末）。

步骤 1：获取 KL-API 中转服务

KL-API 是一个 API 中转平台，支持 OpenAI、Claude、Gemini 等主流模型。关键是国内可以直接访问，响应速度也不错。

去他们官网注册个账号（链接：https://kl-api.top），然后创建一个新的 API Key。你会拿到类似这样的 Key：

kl-xxxxxxxxxxxxxxxxxxxxxxxx

步骤 2：修改 OpenClaw 配置

打开你的 OpenClaw 配置文件，通常是 ~/.openclaw/config.json。

找到 models 部分，改成这样：

{
  "models": {
    "default": {
      "provider": "openai",
      "model": "gpt-5.4",
      "api_key": "kl-你的APIKey",
      "base_url": "https://api.kl-api.top/v1"
    },
    "fast": {
      "provider": "openai", 
      "model": "gpt-5.3-codex",
      "api_key": "kl-你的APIKey",
      "base_url": "https://api.kl-api.top/v1"
    }
  }
}

划重点：

  • base_url 一定要改成 KL-API 的地址
  • api_key 用 KL-API 给你的 Key，不是 OpenAI 的
  • model 名称和官方保持一致，gpt-5.4 或者 gpt-5.3-codex

步骤 3：测试连接

保存配置后，重启 OpenClaw：

openclaw restart

然后测试一下模型是否正常工作：

openclaw chat --model gpt-5.4

如果看到正常的交互界面，说明配置成功。

我测试了一下 GPT-5.4 的代码能力，说实话，真的强。我写了一个需求让它帮我重构一个 Python 项目，它不仅给出了代码，还分析了每个改动的理由，甚至发现了两个潜在的 Bug。

步骤 4：配置 Gemini 3.1 Flash-Lite（可选）

如果你想同时用 Gemini 3.1 Flash-Lite（速度快、价格便宜），可以再加一个配置：

{
  "models": {
    "gemini": {
      "provider": "google",
      "model": "gemini-3.1-flash-lite",
      "api_key": "kl-你的APIKey",
      "base_url": "https://api.kl-api.top/v1"
    }
  }
}

KL-API 的好处是同一个 Key 可以访问多个模型，不用每个平台都注册一遍。

一些使用建议

折腾完这 3 小时，我总结了几条经验：

1. GPT-5.4 适合什么场景？

  • 复杂代码重构和分析
  • 需要多步骤推理的任务
  • 电脑操作自动化（如果你有相关需求）

2. GPT-5.3-Codex 更适合什么场景？

  • 日常编程辅助
  • 快速生成代码片段
  • 性价比更高

3. 关于 KL-API

  • 稳定性还不错，用了几天没遇到超时
  • 计费比官方便宜一些（大概是官方的 7-8 折？）
  • 支持模型挺全的，OpenAI、Claude、Gemini 都有
