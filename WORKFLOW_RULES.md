# 工作流程规则

## 内容生成规则

### 1. 语言要求
- **所有生成的内容必须翻译为中文**
- 包括但不限于：网页、文档、报告、摘要
- 专有名词可保留英文，但需添加中文说明

### 2. 文件管理
- **所有生成的页面文件自动上传到 GitHub 仓库**
- 仓库：https://github.com/kratos-xzm/hw-static
- 使用脚本：`/home/kratos/.openclaw/workspace/push_to_github.sh`

### 3. 链接发布
- **每次生成文件后，必须提供访问链接**
- 格式：`https://kratos-xzm.github.io/hw-static/[文件名]`
- 提醒：GitHub Pages 需要 1-2 分钟部署时间

### 4. 文件命名规范
- 使用有意义的英文名称（便于 URL 访问）
- 格式：`描述_日期.html`（如：`musk_tesla_feed_2026_02_08.html`）
- 主页始终更新 `index.html`
